# Post-v1.0 Roadmap - Yggdrasil Chronicles

This roadmap outlines the evolution of Yggdrasil Chronicles from a technical MVP into a full-featured JRPG.

## v1.1: JRPG Polish & Usability (Completed)
*Goal: Move beyond technical demo to a polished, playable experience.*
- [x] **Title Screen Flow:** Authentic New Game / Continue Game selection.
- [x] **JRPG HUD:** Visual resource bars and cleaner layout.
- [x] **World Interaction:** NPC interaction prompts and player walk animations.
- [x] **Quest Journal UX:** Detailed checklists and completion status.
- [x] **Combat Feedback:** Damage popups, healing indicators, and status clarity.
- [x] **Save/Load Polish:** Clear user feedback on success and failure.

## v1.2: Content Expansion & Systems Depth (Current)
*Goal: Fill the world with meaningful exploration and JRPG systems.*
- [ ] **Deterministic Shop System:** Merchant NPCs with item catalogs and gold-based purchasing.
- [ ] **Inn/Rest Mechanic:** Location-based recovery for HP/MP.
- [ ] **Major Cities:** At least 2 distinct urban centers with unique tilemaps.
- [ ] **Region Expansion:** Sylvan Deep and other new playable areas.
- [ ] **Lore Archives:** In-game library or NPC dialogue trees explaining the world's history.
- [ ] **Expanded Quest Chains:** Multi-step quests with significant rewards and world impact.

## v1.3: Combat Depth
*Goal: Tactical complexity and strategic variety.*
- [ ] **Elemental Affinities:** Rock-paper-scissors damage system (Fire/Water/Earth/Air).
- [ ] **Status Effects:** Poison, Sleep, Paralysis with clear UI indicators.
- [ ] **Row/Formation System:** Frontline vs Backline positioning impact.
- [ ] **Telegraphed Attacks:** Enemies charge powerful moves, allowing for defensive play.

## v1.4: World/NPC Depth
*Goal: Dynamic world feel and reactive storytelling.*
- [ ] **Day/Night Cycle:** Different NPCs and enemies appearing based on time.
- [ ] **Reputation System:** NPC reactions change based on quest outcomes.
- [ ] **Branching Quests:** Choices that permanently alter certain town states.

## v1.5: Release Candidate Polish
*Goal: Final quality assurance and performance optimization.*
- [ ] **Asset Optimization:** Sprite sheet packing and texture compression.
- [ ] **Performance Audit:** Ensuring 60FPS on low-end hardware.
- [ ] **Accessibility Suite:** Screen reader support and color-blind modes.
- [ ] **Final Balancing:** Comprehensive XP/Gold curve adjustment.
